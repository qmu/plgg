# Add PoC 4b: live co-editing preview — the change happens on the preview etc

## Summary

PoC 4b of the plggpress confidence-collection fleet: a new private package `plgg-poc4b-coedit` where the AI's edit happens *on* a live preview — granular find/replace edits land in place (the reloading iframe retired), the changed span animated or diffed, while the same Realtime session keeps talking.

## Key Changes

- New package `plgg-poc4b-coedit`: granular `edit_doc` find/replace tool, a pure applier / span-locator / diff-builder core, and a TEA app split into a gated pure reducer and excluded IO effects.
- Retired PoC 4's reloading iframe and two-process container: a single-process shell renders and patches a live markdown preview client-side; the `plggpress` dependency was dropped.
- Two compared change visualizations — a micro-animation (erase → write) and a before/after diff, toggleable — both driven by one pure diff; `/api/edit` returns the diff so preview and disk agree.
- Portal `poc4b` entry (building, port 5190, `plgg-poc4b.qmu.dev`) plus single-process workload and full fleet wiring.
