# Curry match so tag handlers receive the narrowed box, with CoverageError exhaustiveness

## Summary

This branch strengthens plgg's type system and extends experimental plgg-web with five missing mechanisms: curried `match` for sound union folding with exhaustiveness checking, 401/403 HTTP errors, O(path-depth) compiled route dispatch, binary and streaming bodies, and scoped middleware preventing leaks.

## Key Changes

- Curry `match` so tag handlers receive the narrowed box with typed `.content`, enabling sound union folding and `CoverageError` exhaustiveness checking
- Add `HttpError` 401/403 and a generic `statusError` to close the auth/permission failure-vocabulary gap
- Compile the route table (method index + static exact-map + dynamic walk) for O(path-depth) dispatch instead of O(routes × depth)
- Support binary (`Uint8Array`) and streaming request/response bodies, with platform stream types confined to the seam
- Scope sub-app middleware to its group, eliminating leaks where a guard mounted on `/api` ran app-wide

## Changes

### Added

- 401/403 HTTP error variants (`Unauthorized`, `Forbidden`) and generic `statusError` constructor to close auth/permission failure vocabulary gap
- `Result.mapErr` combinator and `matchResult` case eliminator to core Result type
- `decodeJson`/`encodeJson` codec functions returning `Result` for safe JSON handling
- Binary request support via `bytes: Option<Uint8Array>` field and `bytesResponse`/`streamResponse` builders
- Route-scoped middleware stack to eliminate app-wide leakage and preserve onion composition

### Changed

- `match` curried from single call to two calls (`match(value)(...cases)`) to enable handler narrowing and exhaustiveness checking
- Route dispatch compiled with method indexing and static exact-map for O(path-depth) lookup
- Response body extended to union of `SoftStr | Bytes | Stream` with content-type sniffing and `Content-Length` headers
