# Resolved plgg-bundle's __externals lookup failure that left every plggmatic-example page blank at runtime etc

## Summary

Fixed the plgg-bundle defect that broke any app bundle inlining two or more plgg-family dists: the inlined dist's inner `__externals` table is now rekeyed in step with the outer require rewrite. plgg-bundle 0.0.6 is published; all five plggmatic-example pages render again.

## Key Changes

- Resolved plgg-bundle's `__externals` lookup failure that left every plggmatic-example page blank at runtime (`plgg_1.box is not a function`)
- Added `replaceExternalKey` so an inlined dist's inner externals-table key follows the outer require rewrite in the same step
- Regression spec bundles an app over two emitter-generated dists sharing the inner module key `src/index.ts` and evaluates the emitted bundle end-to-end
- plgg-bundle bumped to 0.0.6 and published; the fix applies at consumer flatten time, so already-published dists need no republish

## Changes

### Added

- `modernize-plgg-bundle` mission: publish-flow DX (parallel preflight, structured output, auto-skipping gate), compiled-dist bin distribution, and the standing plgg-bundle structural debts

### Changed

- `collectModules.ts` rewrites an inlined dist's inner `__externals` table key to the outer module id (`replaceExternalKey`, gated on `isInstalledDist`), with a red-to-green regression spec
- Synced consumer package-lock files to the linked plgg-bundle 0.0.6 (and plgg-kit's lockfile to the 0.0.5-era linked versions it had missed)
