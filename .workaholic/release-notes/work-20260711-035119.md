# Add the plggpress PoC portal and the browser search core PoC etc

## Summary

Opens the plggpress technical-confidence mission and ships its first two artifacts: a PoC portal indexing the six-PoC confidence-collection fleet, and PoC 1 — a browser-side benchmark of from-scratch full-text search against local-model vector RAG on the real guide corpus.

## Key Changes

- PoC 1 measures the plggpress search decision with real numbers: on the 38-file guide corpus, the from-scratch BM25 index costs 252 KB / 19 ms to build; MiniLM vector embeddings cost 1.4 MB (5.7× the corpus) / ~11 s — query-time model download stays visible as a metric
- The PoC portal records the six-PoC plan (question, confidence signal, status, verdict) as typed data with the fleet's port/hostname allocation (5183–5190)
- Both search arms run fully locally — no API key, and the corpus never leaves the machine

## Changes

### Changed
- New private package plgg-poc-portal: the static fleet index served at plgg-poc.qmu.dev (:5183), plus a generic serve-poc.sh workload runner
- New private package plgg-poc1-search: build-time indexer (heading-path chunks, inverted index, MiniLM chunk vectors) and a side-by-side browser app with URL-held queries, a measured metrics table, and a ten-query benchmark, served at plgg-poc1.qmu.dev (:5184)
