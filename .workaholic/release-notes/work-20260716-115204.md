# Export the domain vocabulary as a composable manifestDialect, etc

## Summary

The Domain Manifest vocabulary is now a composable dialect: plgg-ir-manifest exports `manifestDialect` (with `manifestLanguage` derived from it), and plgg-ir-language gains `mapDialect`, the variance seam that lets a consumer compose its own view dialect onto the domain vocabulary in one checked language.

## Key Changes

- `manifestDialect` (name: `domain`) exported from plgg-ir-manifest; `manifestLanguage` derived from it so the two cannot drift
- `mapDialect` added to plgg-ir-language: lifts a dialect to a composition's wider node type (a compile probe proved `Dialect<N>` invariant), with mapped forms keeping their own closed vocabulary
- Consumer dialects add forms only — never extend a domain type — enforced by construction and pinned by tests
- The request's whole quality gate passes as tests: heterogeneous compose with zero casts, cross-dialect references resolving through the composition's scope, duplicate names as composition errors
