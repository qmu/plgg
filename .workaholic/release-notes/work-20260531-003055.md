# plgg-view micro-interactions + security hardening + editorial redesign

## Summary

`plgg-view` grows from a minimal renderer into a micro-interaction-capable Elm-Architecture view library: a focus-safe diff/patch reconciler gains keyed reconciliation, FLIP movement, and a coherent enter/exit/size + sequenced-delete motion lifecycle, styled through a single atomic primitive with SSR critical-CSS extraction and model→URL reflection. The example To-Do app becomes a real demo (modal, toaster, accordion, click-reorder, URL-reflected filter/search, responsive) on that engine. A security review of the from-scratch HTTP/SQL/CI surface lands six fixes, the renderer's next runtime primitives are scoped as four research spikes, and the branch finishes with a warm editorial redesign and refined motion.

## Key Changes

- Keyed reconciliation + FLIP movement and a declarative motion lifecycle (enter/exit/size, sequenced FLIP-delete) on a focus-safe diff/patch renderer
- One atomic styling primitive with SSR `<head>` critical-CSS extraction, plus a typed query codec and model→URL reflection
- A feature-rich example demo: modal, toaster, accordion, click-driven reorder, filter/search-to-URL, responsive layout
- Six security fixes from a full review: SSR CSS-injection XSS, request-body/timeout DoS, fetch redirect credential leak, URL/`on*`/tag sanitization, prototype-safe request lookups, SQL symbol-brand + placeholder invariant, and CI shell-injection hardening
- Four research/design spikes recommending the next primitives (effects/`Cmd`, ref/post-paint, event decoders, reversible size), and an editorial palette + flat UI + refined easing tokens

## Changes

### Added

- Keyed reconciliation (`key()`) + FLIP movement and the deferred-removal exit lifecycle in the client renderer
- Declarative motion: `transition`/`fadeIn`/`fadeOut`/`slideIn` over a `Motion`/`Frame` model and an injectable WAAPI `Play` seam; refined easing tokens (`easeOut`/`easeIn`/`easeInOut`); optional `Motion.delayMs` for sequenced (delayed) motion
- Typed content-model constraints (`Html<Msg, T>`); type-driven inline-style utilities and atomic `css()` extraction
- plgg-router bidirectional typed query codec (`serializeQuery` + `QueryCodec`); plgg-view model→URL reflection (`toUrl`/`historyMode`)
- A `workloads/development` Docker workload to run the example
- Four research/design tickets (recommendations only): ref/post-paint hook, effects & subscriptions, event payload/preventDefault model, reversible size lifecycle
- Example micro-interactions: modal, toaster, accordion, and click-driven reorder

### Changed

- Renderer rewritten as a focus-safe O(changes) diff/patch reconciler
- Styling unified into one primitive (removed the `style_`/`css` split); SSR document inlines critical CSS
- Delete motion reworked: removed row fades out of flow in place, survivors hold then slide up via a delayed FLIP, container height eases shut (replaces the in-flow height collapse)
- Editorial visual redesign: warm ink-on-cream palette, flat/square/borderless UI, serif display + mono eyebrow
- Example client bundle cache-busted (versioned `/main.js` + no-cache); example server port is env-configurable (`PORT`)
- **Security:** CSS selector/prop/value escaped to close a `</style>` SSR XSS; node adapter bounds request bodies (413) and sets socket/header/request timeouts; both fetch seams use `redirect: "manual"` with a typed `RedirectError`; URL schemes / `on*` attribute names / SSR tag names sanitized identically in SSR and client; request map lookups guarded with `Object.hasOwn`; `Sql` symbol-branded with a placeholder/param invariant; CI `start-pull-request` shell-injection fixed, unused `id-token` dropped, gems pinned
