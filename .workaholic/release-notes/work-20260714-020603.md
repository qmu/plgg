# PoC 4 proven, plgg-md on plgg-parser, `plggpress dev`, and PoC 4c

## Summary

The plgg family stopped hand-rolling what it already owns. PoC 4 is concluded from the
developer's live judgment; plgg-md's markdown grammar moved off regex line-scanning onto
plgg-parser combinators with **byte-identical** output; plggpress gained a first-class
`dev` command that deletes every consumer's hand-wiring; and PoC 4c — the developer's own
idea, raised while judging — is built and serving, awaiting its live verdict.

## Key Changes

- **PoC 4 is `proven`.** Judged live on the fixed build: a Japanese conversation edited an
  English document *in English*, the edit landed, the page hot-reloaded, and the Realtime
  session kept talking. `plgg-poc.qmu.dev` now shows five proven PoCs.
- **plgg-md renders through combinators.** Both parsers (1028 lines of regex line-grammar)
  are plgg-parser grammars; the guide and strategy corpora render **byte-identically**.
- **`plggpress dev`.** A docs repo whose only dependency is plggpress now runs
  `plggpress dev` and gets hot reload — no `bundle.config.ts`, no `devEntry.ts`, no
  bundler devDependency. Both `packages/guide` and `../strategy` deleted theirs.
- **PoC 4c serves at :5198** — 4b's animated in-place edit on PoC 4's real rendered site.
- **Three bugs fixed:** a CRLF markdown source parsed as nothing but paragraphs (a
  Windows-authored `.md` rendered as mush); an unhandled `fs.watch` error killed the dev
  server (it had killed the guide container twice, unnoticed, for hours); a build caused
  spurious browser reloads.
