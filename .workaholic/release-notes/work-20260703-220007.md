# plgg-auth: a from-scratch OpenID Connect identity provider on plgg

## Summary

plgg-auth is a from-scratch OpenID Connect identity provider built on plgg with zero third-party runtime dependencies (WebCrypto + Node built-ins only). It delivers the full authorization-code flow with mandatory S256 PKCE, an app-supplied AuthStore persistence seam, refresh-token rotation, and signing-key rotation.

## Key Changes

- Vendor-neutral signing foundation: the Jose layer (base64url, JWK/JWKS with RFC 7638 thumbprint `kid`s, JWS RS256, JWT validation) built in-house on WebCrypto, pinned to the RFC test vectors and cross-checked against `node:crypto`
- AuthStore seam inversion: the OP protocol is persistence-agnostic; an in-memory driver and a plgg-sql driver pass an identical shared contract spec
- Security boundary as a type: `/authorize` folds into a pure `AuthorizeOutcome` union so open-redirect prevention is a domain type, and the OP never touches passwords (app-owned login seam)
- Defense-in-depth: refresh-token rotation with family-revocation reuse detection, signing-key rotation (active â†’ retiring â†’ retired), and token values stored only as SHA-256 hashes
