# Dogfooding fixes across plgg core, CLI, md, plggpress, and fetch

## Summary

Six dogfooding tickets from rebuilding real codebases on the plgg family, each closing a gap that had forced a seam back to a raw or vendor path. Fresh full check-all is green; no new dependencies.

## Key Changes

- plgg core: point-free `mapResult` now keeps its error type, plus a new `bracket` resource-scope combinator.
- plgg-fetch gains request timeouts, binary/streaming response reads, multipart bodies, and bearer/versioned auth headers.
- plggpress resolves extensionless config imports and adds `srcExclude` / `linkIgnore`.

## Changes

### Added
- plgg `bracket` (guaranteed release on Ok/Err/throw/reject) and point-free-safe `mapResult`.
- plgg-cli `runSingle` (bare-flag CLIs) and typed `optionList` / `optionInt` readers.
- plggpress `srcExclude` and `linkIgnore` site config, plus extensionless relative TS config imports.
- plgg-fetch `timeoutMs`, `readAs` bytes/stream readers, multipart builder, and auth header helpers.

### Changed
- gateStamp computes its tree digest lock-free (no more `git stash create` flake) with retry.
- plgg-md frontmatter refuses nested sequences with an actionable message and documents the supported alternative.
