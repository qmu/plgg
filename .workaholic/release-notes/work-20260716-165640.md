# Add plgg-search, the browser-shippable FTS core ported from PoC 1

## Summary

Integration 1/8 of the post-PoC chain: PoC 1's proven browser-side search core is now the production `plgg-search` package — shared latin+CJK tokenizer, heading-path markdown chunker, inverted-index builder, and from-scratch BM25 — with the tokenization strategy carried inside the index so build and query can never mismatch.

## Key Changes

- New `plgg-search@0.0.1` under the `domain/{model,usecase}` layout, dependency set exactly `plgg`
- Browser-shippability pinned by spec: no `node:` imports, nothing server-shaped, anywhere in production source
- The vector arm deliberately left behind per PoC 1's verdict; semantic search stays server-side in plgg-cms
- Dependency root in place for integration 2 (static `fts.json` + sitemap) and 3 (reader agent)
