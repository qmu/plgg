# Add plgg-parser combinator library; migrate plgg-highlight off ts.createScanner

## Summary

A new zero-new-dependency parser combinator library, `plgg-parser`, is built purely on plgg, and it retires the last `typescript` dependency from the published `plgg-highlight` surface: the highlighter's `ts.createScanner` tokenizer is replaced by an in-house plgg-parser TS grammar, with its public contract (signature, never-throw, exact-source round-trip) unchanged.

## Key Changes

- Add `plgg-parser`: data-last `Parser<A, S>` combinators on plgg, with a threaded user-state slot for context-sensitive grammars and a TS-lexer proof-of-value demo
- Migrate `plgg-highlight` tokenization from the TypeScript compiler's `ts.createScanner` to a plgg-parser grammar
- Drop `typescript` from `plgg-highlight`'s peerDependencies (kept as a devDependency for type-checking only)

## Changes

### Added
- `packages/plgg-parser` — a generic parser combinator library depending only on plgg (Result/Option/Box/InvalidError). Parsers are data-last functions `Parser<A, S> = (ParseState<S>) => Result<Parsed<A, S>, InvalidError>` composed with pipe/flow; `ParseState<S>` carries a user-state slot for context (e.g. regex-vs-division). Full combinator set (map/andThen/seq/left/right/or/many/many1/optional/between/sepBy/sepBy1/lookahead/notFollowedBy/lazy/getUserState/setUserState/run) plus primitives; a TS-lexer demo proves exact-source round-trip, nested `${}` interpolation, regex-vs-division, and unterminated-at-EOF plain degradation

### Changed
- `plgg-highlight` tokenizes with an in-house plgg-parser TS grammar instead of `ts.createScanner`; the 9-variant `TokenKind` taxonomy, the `tokenize(code): Token[]` signature, the never-throw guarantee, and the exact-source round-trip are all preserved (legacy `tokenize.spec.ts` kept byte-identical as the parity net, with new edge-case specs added)
- `plgg-highlight` now depends on `plgg-parser`; `typescript` moves from a peerDependency to a devDependency only, so highlighting adds no dependency to a consumer's install
