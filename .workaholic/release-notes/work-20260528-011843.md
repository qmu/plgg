# Combine plgg-view + plgg-sql + plgg-http-client into one runnable example, lead the README with the picture

## Summary

The three sibling POCs (`plgg-view`, `plgg-sql`, `plgg-http-client`) now meet in one full-stack `src/example` package — `plgg-http-router` serves SSR HTML rendered by `plgg-view` from rows fetched through `plgg-sql`, the browser hydrates the same VNode tree, and a `plgg-http-client` node script consumes the same JSON API. The root `README.md` opens with a paired snippet (server `POST /articles` `proc` chain, client `GET /articles` `pipe` branching into JSX) so the first thing a reader sees is what "web development as one pipeline" looks like in code.

## Key Changes

- `src/example` extended into one end-to-end demo: SSR + JSON + CSR + node-script client over a seeded `:memory:` SQLite
- Cross-package seams fixed at the source (NodeNext nested `exports.types` on `plgg-sql` + `plgg-http-client`, raw-row wire shape so `Option`/`Time` survive JSON, `Name` as `SoftStr` so the view renders it without unwrapping)
- Root `README.md` opens with "Web development as one pipeline": server `proc` chain and client `pipe`/`matchResult`/`matchOption` branching into JSX

## Changes

### Added

- `src/example/src/db/open.ts`: `node:sqlite` `Db` seam, seed, and `compactRow` (drops null-valued columns so a NULL `memo` decodes to `None`)
- `src/example/src/server/app.ts`: SSR page + `/api/articles` JSON + `/client.js` CSR bundle routes, one `proc` chain each
- `src/example/src/client/main.ts`: `plgg-http-client` demo consuming `/api/articles`, folding `ClientError` exhaustively by named ADT pattern
- `src/example/src/db/open.ts` seed: three articles (one with a NULL `memo`) exercising the `Option` path end-to-end
- Root `README.md`: "Web development as one pipeline" section with paired server/client snippets

### Changed

- `src/example/src/App.tsx`: data-driven (`{ articles }` prop) instead of a hardcoded `FEATURES` list
- `src/example/src/ssr/server.ts`: DB-backed (opens the seeded `Db`, threads it into `buildApp`)
- `src/example/src/csr/client.tsx`: hydrates from `/api/articles` via native `fetch` (keeps `node:http` out of the browser bundle)
- `src/example/src/modeling/Article.ts`: `Name` as `SoftStr` so the view renders the name as text directly; `asArticles` decoder shared by SSR, hydrate, and `plgg-http-client` paths
- `plgg-sql` + `plgg-http-client` `package.json`: nested `exports.types` under each subpath so NodeNext consumers resolve types correctly
