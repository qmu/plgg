# Add PoC 4 (agent file edits with live hot reload over a surviving Realtime session) and the root .env credential convention etc

## Summary

PoC 4 of the plggpress confidence-collection fleet: a browser agent's edits land on a git-ignored copy of the guide through the dev server, the doc iframe hot-reloads, and the same OpenAI Realtime session (voice and typed text) survives — proven headless and served live at `plgg-poc4.qmu.dev` on `gpt-realtime-2.1`. Live judging surfaced and fixed two bugs (language lock-in; edit corruption from feeding the model a lossy chunk reconstruction), and reframed the next question as the co-editing *experience* (PoC 4b, queued). The branch also establishes a single git-ignored repo-root `.env` credential convention sourced by the canonical `serve-poc.sh` runner.

## Key Changes

- New `packages/plgg-poc4-edit`: granular edit seam (`POST /api/edit`) behind one authoritative `resolveEditPath` guard + realpath containment, a guarded raw-markdown read seam (`GET /api/doc`), iframe-isolated hot reload so the WebRTC session survives, voice + typed input over one session
- Fixed two live-judging bugs: the assistant now honors an explicit language switch (default stays the document's language), and the editing model is fed the real file bytes so edits round-trip as clean markdown (retired the lossy `docTextOf`)
- Pinned the Realtime model to a spec-pinned `gpt-realtime-2.1` constant feeding both the server mint and the browser SDP exchange
- Established the repo-root `.env` credential convention: `serve-poc.sh` sources it (caller env wins, injection-proof), a committed `.env.example`, and a companion worktree-copy protocol in the workaholic plugin

## Changes

### Added

- `packages/plgg-poc4-edit` (private PoC) with an exhaustively spec'd edit-path security boundary, `/api/edit` write seam, `/api/doc` raw-read seam, and in-process FTS index refresh

- The Realtime model is pinned via one `REALTIME_MODEL` constant (both mint and SDP), replacing the drifting `gpt-realtime` alias
