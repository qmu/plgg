# Add four foundation combinators to eliminate repetitive type scaffolding etc

## Summary

Unified plgg's foundation with four new combinators, applied branded types only at untrusted boundaries, extracted the plggmatic framework with plggpress as its thin consumer, added a hot-reload dev server to plgg-bundle, documented all 18 packages, fixed two path-traversal vulnerabilities, and thickened plggmatic into a full facade that collapses plggpress's dependencies from 7 packages to {plgg, plggmatic}.

## Key Changes

- Added four foundation combinators (defineVariant, refinedBrand, Ord/compare, foldThrown) eliminating ~50 sites of repetitive type scaffolding
- Delivered the plggmatic framework extracted from plgg-press, which is now a thin consumer renamed plggpress with a qmu.co.jp theme
- Implemented the plgg-bundle dev server with real code hot-reload via module-runner
- Established branded-type boundaries at untrusted edges (HttpStatus, TenantId, Int/Float) while keeping author-facing APIs plain
- Hardened plgg-db-migration against path traversal and completed guide/README coverage for all 18 packages
- Thickened plggmatic into a re-export facade over plgg-view, plgg-server/plgg-http, plgg-md and plgg-highlight; plggpress now depends on {plgg, plggmatic} alone

## Changes

### Added

- plgg-cli toolkit for building command-line program wrappers
- defineVariant combinator collapsing the four-fold Box-variant scaffold

- Refined integral ids, counts and indices to Int; CSS opacity to Float; HTTP status to a typed HttpStatus boundary
- Resolved the SoftStr/KebabCase sweeps under principle (a): author-facing strings stay plain
- defineSite takes a typed author-facing argument
- Collapsed async-Result ladders onto proc, folding Defect at the CLI edge
- Reskinned the docs site to the qmu.co.jp theme (tokens, typography, sidebar-first shell)
- Reimplemented plgg-press as a thin plggmatic consumer and renamed it plggpress
- plgg-bundle falls back to native import() for unresolved bundle ids
- Hardened TenantId to a safe path segment and sanitized newMigration names (path-traversal fixes)
- plggmatic re-exports the full mid-library surfaces with plggmatic/ssg and plggmatic/style subpath mirrors
- plggpress imports everything through the plggmatic facade; its dependency list collapsed from 7 packages to 2
