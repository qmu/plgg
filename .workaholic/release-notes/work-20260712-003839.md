# Build the plgg-ir package family: typed S-expression IR for AI-generated Domain Manifests

## Summary

The entire plgg-ir mission lands in one branch: three new packages implementing the restricted, typed, Lisp-style intermediate representation that separates AI-generated intent from deterministic Web-system generation. An LLM agent emits a Domain Manifest; the toolchain statically verifies it end to end and normalizes it into a deterministic canonical IR. The design §39 acceptance scenario is green and all 12 mission acceptance items are ticked.

## Key Changes

- `plgg-ir-syntax`: position-aware S-expression parsing/printing with recovery-based coded diagnostics and the property-tested round-trip law
- `plgg-ir-language`: the reusable static language framework — semantic types with nominal domain distinctness, typing-rules-as-functions, two-phase declare/analyze scopes, idempotent normalization with a canonical serializer, dialect composition
- `plgg-ir-manifest`: the full Domain Manifest dialect — entities, relations, aggregates, projections, policies, views, actions, derivations — with layered boundary diagnostics, deny-by-default authorization as a compile error, and a machine-checked dependency graph with topological update planning
- The §39 end-to-end scenario compiles into the canonical IR; canonical output is a compile fixpoint; the unauthorized-edit and cross-aggregate rejections hold as diagnostics
- Comprehensive end-to-end guide at `docs/plgg-ir/guide.md`

## Changes

### Added

- Three packages (`plgg-ir-syntax`, `plgg-ir-language`, `plgg-ir-manifest`) with 175 tests, per-package >90% coverage (100% for the two lower layers), wired into build/install/check-all/test scripts and the README hierarchy; vendor-boundary conformant unexempted
- `compileManifest` — the single consumer entry point returning the resolved `Module` model plus deterministic canonical text, with every diagnostic of every stage in one list
- `derivedUpdateOrder` / `updatePlanFor` — pure update-planning API over the compiled model (the §13 chain `order-item change → total → tax`)
- `docs/plgg-ir/guide.md` — grammar, framework concepts, complete vocabulary, diagnostic-code table, safety-principle enforcement map, known deviations
