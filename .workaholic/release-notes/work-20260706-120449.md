# Extract the plggmatic design system into its own repository etc

## Summary

The plggmatic demos grew into DevDesk, a full business-management example app, and the Pragmatic design system moved to its own repository. plggpress split into a slim SSG plus the new plgg-cms package, npm publishing became a developer-driven pre-merge gate, and the plgg CLIs became registry-consumable.

## Key Changes

- Extracted the plggmatic design system into its own repository via an Agent-Teams trip, leaving the reusable engine in the plgg family
- Split plggpress into a slim SSG and the new plgg-cms dynamic CMS package, then retired the plgg-ui and plgg-domain package boundaries into plgg-cms
- Grew Demo 1 into DevDesk, a contract-dev business-management app with a typed URL-state codec, pure update, and a written security assessment

## Changes

### Added
- A /demo section with three isolated pillar demos: pane alignment, color scheme, and scheduler query with derived URL codec
- Demo 1 security assessment report with href-sink regression specs
- Dynamic scheduler Sources so app records live in the Model
- Pragmatic screen-transition and input-field model specifications
- plgg-md raw-HTML passthrough and an injectable heading slugger

### Changed
- npm publish is now a developer-driven, pre-merge, bump-only ship gate
- Demo 1 rebuilt as DevDesk with generic record sections, a typed URL codec, and a module split
- plggpress split into a slim SSG plus plgg-cms; Prag UI, content, and domain consolidated into plgg-cms
- plgg-bundle, plgg-test, and plggpress are installable from the npm registry; plgg-bundle inlines published dependencies for standalone apps

### Removed
- The plggmatic cluster left the monorepo for its own repository; the plgg-ui and plgg-domain package boundaries were retired
