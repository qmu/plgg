# Errors as tagged data + precise proc inference, plus an SSG generator etc

## Summary

Redesigned plgg's error model from `Error` subclasses to pure tagged data with a
single `Defect` bottom, made the `Result` effect family carry the precise error
type through `proc`, hardened the result against review findings, and shipped a
static site generator on the new foundation.

## Key Changes

- Errors are now pure tagged data (`Box` unions), not `Error` subclasses — the no-class rule is universal.
- `proc` infers the exact error union (`E₁ | … | Defect`), locked by a type-level assertion.
- New `plgg-server/ssg` static site generator (crawl routes via `handle`, write directory-index HTML).
- Review-driven fixes: a `printPlggError` crash, `bind`/`tryCatch` minting raw `Error`, and a `Defect` cause lost through JSON.

## Changes

### Added

- `Defect` bottom error, `toError`/`panic` interop, and the `plggErrorMessage`/`matchPlggError`/`resultErrorMessage` accessors.
- `plgg-server/ssg`: `generateStatic` renders explicit routes to static directory-index HTML, with `node:fs` confined to one node-only entry.

### Changed

- Migrated the error model to tagged `Box` data (deleted the `BaseError`/`Exception` class hierarchy); `proc` now infers the precise per-step error union instead of `unknown`.
- Hardened the foundation: `printPlggError` cycle guard, `bind`/`tryCatch` no longer mint raw `Error`, `isPlggError` shape guard, a serializable `Cause`, and `cast` throw-capture.
