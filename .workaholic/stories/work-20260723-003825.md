# plggpress: column layout, config theme, persistent dev server + live-edit bridge

_Mission: plggpress-column-layout-and-voice-ai-editing (created + driven this session by `/monitor`, run monitor-20260723-011758). Two waves; 4 of 6 tickets landed green._

## 1. Overview

New mission moving plggpress onto plggmatic's column-oriented horizontal layout
and building the substrate for conversational (voice) editing in dev. This branch
kicks off the mission (statement, strategy `plgg-horizontal-orientation-ui-stack`,
6-ticket plan) and lands the first four tickets. The voice-realtime assistant and
its guide page remain deferred with the substrate now in place.

## 2. What changed

**004000 — adopt plggmatic column layout.** plggpress now depends on the
`plggmatic` package and renders its docs nav as the plggmatic `pm-row`/`pm-col`
column strip (row/column combinators + `vp-*` geometry hooks via `style_` parts,
no `pm-*` name overrides — so no theming smell). The vendored `themeSupport`
Style/Component/Meta tree was deleted; `styleEntry` reduced to
`export * from plggmatic/style`. `page.spec` asserts the strip structure, that
drilling a section adds a column (home = 2 cols, in-section = 3), width-invariance,
and a grayscale scheme.

**004010 — config-driven theme.** `SiteConfig` gained an optional `theme` field
validated via plggmatic `pragmaticThemeWithPalette`; `baseCss`/shell became
functions of the resolved theme (`resolveTheme` — override else the B&W default).

**004020 — persistent dev server.** A dev-only, plggpress-owned persistent
`node:http` surface (`startDevServer` + `makeReloadHub` SSE broadcast hub +
`devWeb` injector + reload client). The SSE channel survives rebuilds because the
server process holds the hub across content edits; an edit pushes a reload frame
without dropping the stream. Delivered with a start-server/connect-SSE/edit-file/
observe-reload integration test.

**004030 — live-edit bridge.** `POST /__plggpress_patch` promotes PoC 4b's proven
pure core (`applyEdits` exactly-once locator + `resolveEditPath` lexical guard)
into production, plus fs realpath-containment and atomic temp+rename write, then
`hub.notify`. Integration test covers disk-edit + hot-reload + channel-stays-
connected + out-of-content-dir rejection (400, writes nothing) + 400/404/422 misuse.

## 3. Outcome

own SSE reload channel and a guarded live-edit endpoint — the full substrate a
voice assistant will drive. 4/6 acceptance ticked.
