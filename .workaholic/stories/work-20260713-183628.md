## 1. Overview

Bumped `plggpress` from 0.0.3 to 0.0.4 and published it to npm so plggmatic's documentation site can consume the current source. The registry was stale at 0.0.3 while `packages/plggpress/src` had drifted; npm cannot republish a version, so delivering the current plggpress required the bump and a developer-run publish.

**Highlights:**

1. `plggpress@0.0.4` published to npm (dist-tag `latest`), unblocking plggmatic's docs to depend on `^0.0.4`
2. Version bump only — no plggpress source change rides on this branch; the publish ran through the canonical `scripts/publish-npm.sh`

## 2. Motivation

plggmatic's documentation depends on plggpress from the registry and needs the current source, but `plggpress@0.0.3` was already published and `packages/plggpress/src` had moved on since. Because npm forbids republishing a version, the only way to ship the current plggpress was to bump to 0.0.4 and republish — a release-enablement chore kept off the PoC 4 branch and driven from its own branch off main.

## 3. Changes

This branch is a single release-enablement chore.

### 3-1. Publish plggpress (bump 0.0.3 → 0.0.4) for plggmatic docs ([cb30f016](https://github.com/qmu/plgg/commit/cb30f016))
