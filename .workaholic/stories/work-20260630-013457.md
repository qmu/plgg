## 1. Overview

This branch retires the plgg family's last third-party tooling and replaces it with in-house, plgg-native tools — then sheds the dependencies those tools left behind. It lands three interlocking arcs: (1) **plgg-press**, a from-scratch VitePress replacement (markdown, syntax highlighting, static-site generation, theme, dev server) that the guide now runs on; (2) an **in-house test DOM** in plgg-test that retires happy-dom monorepo-wide; and (3) a **third-party dependency purge** (happy-dom, api-extractor, typedoc, typedoc-plugin-markdown, vitepress, vite) that leaves the entire repo's external surface at just `typescript` + `@types/node`. 47 commits, 234 files. The full `check-all` gate is green: `gate-vite` ✓, `gate-happy-dom` ✓, `tsc-plgg` (all packages) ✓, `test-plgg` **465 passed / 0 failed**.

**Highlights:**

1. Built **plgg-press** + `plgg-md` + `plgg-highlight` from scratch on plgg and migrated the guide off VitePress entirely (vitepress + vite fully removed; no-vite gate added).
2. Built plgg-test's **own from-scratch DOM** test environment (node graph, bubbling event dispatch, CSS selector engine, window/history) — class-free, no `as`/`any`, zero new deps — and removed happy-dom everywhere (no-happy-dom gate added).
3. **Purged** api-extractor (dead) and typedoc + typedoc-plugin-markdown (by deleting the thin auto-generated API reference); the monorepo's only remaining third-party libraries are now `typescript` and `@types/node`.
4. Extensive **plgg-press theme + responsive** work: VitePress-parity visuals, dark mode, sidebar hierarchy, mobile nav, mid-range breakpoint, code-block overflow fix, `no-store` dev headers.

## 2. Motivation

The repo's standing vendor-neutrality / smaller-better posture: implement it ourselves and depend on as little third-party code as possible. VitePress (and its vite/typedoc-theme transitive weight), happy-dom, api-extractor, and typedoc were the remaining external tools. Each is replaceable by a plgg-based tool or removable outright — and doing so proves the family's own primitives are strong enough to build real tooling (docs pipeline, test DOM) on.

## 3. Changes

### 3-1. plgg-press: an in-house VitePress replacement
New packages built on plgg: **`plgg-md`** (frontmatter splitter + block/inline tokenizer folding to an `Html` AST with slug parity), **`plgg-highlight`** (zero-dep TS/TSX highlighting via `ts.createScanner`), and **`plgg-press`** (the SiteConfig contract + href helper + CLI; a `discoverPaths → renderMarkdown → highlight → theme → Ssg` static build; a `node:http` + `fs.watch` dev server with a dev-only SSE live-reload; anchor-aware build-time dead-link checking; and the document-shell/nav/sidebar/home/callout theme). plgg-view gained a content-slot builder and the Markdown/theme tag set. The guide was ported to a typed `site.config.ts`, VitePress was removed (depend only on plgg-press), and `deploy-guide.yml` / `serve-guide.sh` / `gate-vite.sh` were rewired; a Docker/podman dev container builds sibling dists after the bind mount.

### 3-2. plgg-press theme + responsive polish
VitePress-parity visual overhaul; light/dark switch with class-based syntax tokens; sidebar section hierarchy, spacing, and carets; CSS-drawn hamburger + sun/moon icons (no glyph fonts). Responsive: a mid-range breakpoint, fluid `clamp()` hero, mobile nav folded into the ☰ panel in normal flow (no overlay), and a fix for horizontal overflow on wide code blocks. The dev server now sends `Cache-Control: no-store` so a CDN/tunnel never serves stale inlined theme CSS.

### 3-3. In-house test DOM (retire happy-dom)
plgg-test grew its own DOM environment under `src/Env/Dom/`: a mutable node graph, event dispatch with correct `target` + bubbling to document/window, a small CSS selector engine, navigation-inert window/history/location, `DOMRect`/`MouseEvent`, and `instanceof` via `Symbol.hasInstance`. Class-free (type + factories, no `this`), no escape hatches, no new deps. The directive token became `// @plgg-test-environment dom`; the consumer specs (plgg-view, example) migrated; happy-dom was removed from all three package.jsons and every lockfile; `scripts/gate-happy-dom.sh` guards reintroduction.

### 3-4. Third-party dependency purge
Removed the dead `@microsoft/api-extractor` devDep (plgg-test emits its own `.d.ts`). Removed `typedoc` + `typedoc-plugin-markdown` by deleting the auto-generated API reference outright (the per-symbol pages were thin next to the hand-written package guides) — generator, config, golden check, generated `api/**`, site-config splice, nav leaf, and content `/api/` links all removed.

## 4. Outcome

The guide is dependency-free (only the `plgg-press` workspace link). The whole monorepo's third-party surface is `typescript` + `@types/node`. Two new gates (`gate-happy-dom`, and the existing `gate-vite` extended) lock the removals in, shared verbatim between `check-all.sh` and CI. The in-house test DOM is exercised by ~275 consumer specs plus its own unit suite; the guide builds to 25 pages with a green link check.

## 5. Historical Analysis

Continues the vendor-neutrality line of prior branches (the vite purge, the dotenv/happy-dom-leftover pruning). This branch finishes the job: where earlier work removed *unused* deps, this removes the *used* ones by building in-house replacements (plgg-press, the test DOM) or deleting the feature (API reference). The "build it ourselves on plgg" thesis is now demonstrated at the scale of a full docs toolchain and a test-runner DOM.

## 7. Successful Development Patterns

- **Prove the primitive by building real tooling on it** — a docs pipeline and a test DOM, both on plgg, no new deps.
- **Gate every removal** — a canonical grep gate shared local+CI so a shed dependency can't creep back.
- **Verify UI in a real browser** — responsive/theme fixes were measured (viewport, `scrollWidth`) and screenshotted, not asserted blind.
- **Delete beats replace** when a feature earns its keep poorly (the API reference).
