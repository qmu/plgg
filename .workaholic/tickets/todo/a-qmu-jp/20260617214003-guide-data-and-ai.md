# Guide T7 — Data & AI: plgg-sql, plgg-foundry (and plgg-kit)

## Overview

Document the data and AI-orchestration packages: `plgg-sql` (functional SQL
builder + typesafe row mapping) and `plgg-foundry` (LLM-composed workflow
orchestration). Briefly cover `plgg-kit` (vendored dependencies) so the family
coverage is complete.

## Key Files

- `packages/guide/packages/{plgg-sql,plgg-foundry,plgg-kit}/*` (new).
- `packages/plgg-sql/` - the SQL builder, parameterization, `SqlError` (now `Box`
  data), `Mapping` (`decodeRows`/`asRow` typesafe mapping), the `Db` model.
- `packages/plgg-foundry/` - `Alignment`/`Foundry`/operations (the register-machine
  op set), `Switcher`, the LLM `operate`/`blueprint` flow, the LLM vendor
  adapters.
- `packages/plgg-kit/` - vendor dependency surface.
- The three `README.md`s.

## Implementation Steps

1. **plgg-sql** — building parameterized queries functionally, executing via the
   `Db` seam, and typesafe row decoding (`decodeRows` + `cast`-based `asRow`);
   `SqlError` as tagged data and how it folds through `Result`/`matchResult`.
2. **plgg-foundry** — the model: an LLM composes an execution plan (`Alignment`)
   from a register-machine set of operations (a `Foundry`); document
   `makeSwitcher`/operations, `operate`/`blueprint`, and the vendor-adapter
   boundary (Anthropic/Google/OpenAI). Use the `ProfileFoundry` example.
3. **plgg-kit** — a short page: what it vendors and why a plgg project depends on
   it; not an API surface to teach in depth.
4. Cross-link `plgg-sql`/`plgg-foundry` error pages back to the core error-model
   page (T4) — both define domain `Box`-union errors on the same idiom.

## Considerations

- **`SqlError` is now `Box` data** (migrated this cycle) with a serializable
  cause — document the current shape, and note the future shared "boundary
  error" primitive as a forward pointer. (`packages/plgg-sql/src/Db/model/Db.ts`)
- **plgg-foundry has skipped tests / is study work** — be clear about its
  maturity so readers calibrate; it is the most experimental package.
- Keep the LLM-vendor section vendor-neutral in framing (the adapter boundary is
  the point) — `standards:operation`.
- Pull examples from the `plgg-sql`/`plgg-foundry` specs and `Example/` modules.
