# U2 — Migrate `example` from vitest to plgg-test

## Overview

Migrate the `example` package's tests from vitest to `plgg-test`. This
is the smallest package (1 spec file, ungated coverage) and the
leaf-first starting point of the per-package migration wave.

**Trip Origin:** `.workaholic/trips/replace-vitest-with-plgg-test/designs/design-v2.md`
§2 (migration recipe) + §7 (U2, leaf-first).

`example` is ungated (no vitest threshold, `coverage: { all: true }`) —
it gets NO `plgg-test.config.json` (a missing config = ungated, matching
its prior behavior).

## Key Files

- `packages/example/src/app.spec.ts` — the one spec to rewrite.
- `packages/example/package.json` — swap `test`/`test:watch`/`coverage`/
  `coverage:watch` scripts to the plgg-test idiom; remove `vitest` +
  `@vitest/coverage-v8` devDeps; add `"plgg-test": "file:../plgg-test"`.
- `packages/example/vite.config.ts` — remove the `test:` block and the
  `/// <reference types="vitest" />` line; keep `build`/`resolve`/
  `plugins`.

## Implementation Steps

1. Rewrite `app.spec.ts`: `import … from "vitest"` →
   `import … from "plgg-test"`; every `expect(x).matcher(y)` →
   `return check(x, matcher(y))` (each body MUST return its assertion);
   `.not.` → `not(...)`; `await expect(p).resolves.toBe(y)` →
   `return check(await p, toBe(y))`; `assert(isOk(r))` narrowing →
   `okThen`/`shouldBeOk` etc.
2. Swap `package.json` scripts to: `test`:
   `tsc --noEmit && plgg-test src`; `test:watch`: `plgg-test src --watch`;
   `coverage`: `tsc --noEmit && plgg-test src --coverage`;
   `coverage:watch`: `plgg-test src --coverage --watch`. Remove vitest
   devDeps; add `plgg-test`.
3. Clean `vite.config.ts` (drop `test:` block + vitest triple-slash).
4. Format with the package's `.prettierrc.json` (printWidth 50).

## Considerations

- **Definition of done**: `scripts/test-example.sh` green AND
  `scripts/test-watch-example.sh` confirmed to start/re-run. No coverage
  gate (ungated).
- No `as`/`any`/`ts-ignore`; do not copy any cast pattern from the
  reference `plgg` specs.
- Every rewritten body must RETURN its `check(...)`/`all([...])` — the
  runner fails a non-returning body.
