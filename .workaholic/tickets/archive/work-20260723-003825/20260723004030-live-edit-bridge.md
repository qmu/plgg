# Live-edit bridge: patch the open markdown on disk, hot-reload in place

## Overview

Integrate the proven `plgg-poc4c-livesite` edit bridge into production
plggpress: a tool call patches the open markdown source file on disk,
and the rendered page hot-reloads to show it while the live session
(the plggpress-owned client channel from the dev-server ticket) stays
connected — no full navigation, no session drop. This is the editing
substrate the voice assistant will drive; here it is exercised by a
direct patch endpoint (no AI yet).

## Key files

- `packages/plgg-poc4c-livesite/src/` — `bridge.ts`, `patchClient.ts`,
  `inject.ts`, `reloadPolicy.ts`, `spanMap.ts` — the proven artifacts
  to integrate (source-range → file patch → targeted reload).
- `packages/plggpress/src/router/pressRouter.ts` — the render path the
  bridge patches back into.
- The persistent dev-server surface from
  `20260723004020-persistent-dev-server-surface.md`.

## Approach

- Port the bridge as a plggpress dev-server endpoint: accept a typed
  patch (path + span/replacement) against a source `*.md`, write it to
  disk through the plgg file layer, and push a targeted reload over the
  plggpress-owned channel.
- Reuse `spanMap`/`reloadPolicy` so a patch reloads only the affected
  page and keeps the session; the edited page reflects the change on
  the next render (single render path unchanged).
- Guard: the bridge is dev-only and only writes under the configured
  content dir (no path escape); reject patches outside it with a named
  error.

## Quality Gate

- **Acceptance:** an integration spec opens a page over the dev
  channel, posts a patch to the bridge, and asserts (a) the source
  `*.md` on disk now contains the edit, (b) the page hot-reloads to
  show it, (c) the client channel stayed connected throughout. A patch
  targeting a path outside the content dir is rejected.
- `scripts/tsc-plgg.sh` clean; `./scripts/check-all.sh` green; >90%
  coverage; no `as`/`any`/`ts-ignore`; Prettier `printWidth: 50`.

## Policies

- `workaholic:implementation` / type-driven — the patch is a typed,
  Result-cast command; the out-of-content-dir rejection is a checked
  boundary, not a defensive branch.
- `workaholic:operation` — the session survives the edit; reload is
  targeted, the channel self-heals.
