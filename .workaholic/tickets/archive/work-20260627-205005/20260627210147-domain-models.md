# Define the `plgg-db-migration` domain models

## Overview

Define the pure domain vocabulary (types, constructors, branded casters) that
the rest of the package is built from. The vocabulary is the Architect's model,
adopted as canonical, with each term defined only in terms of the ones above it.
No I/O — pure types + validation, fully unit-testable.

## Policies

- `workaholic:implementation` / `policies/directory-structure.md` — models in
  `src/domain/model/`, one concern per file with an `index.ts` barrel.
- `workaholic:implementation` / `policies/coding-standards.md` — no
  `as`/`any`/`ts-ignore`/`null`/`enum`/`class`; `type` over `interface`,
  `Readonly`, closed literal unions; Prettier printWidth 50.
- `workaholic:implementation` / `policies/type-driven-design.md` — branded
  `Version`/`TenantId`; `down` as `Option<SoftStr>` (never empty string);
  `Box`-tagged `MigrationError`; `Dialect` a closed union folded by exhaustive
  `match` so a fourth engine is a compile error.
- `workaholic:implementation` / `policies/functional-programming.md` — casters
  return `Result<_, MigrationError>`; immutable data, no `throw`.

## Key Files

- `packages/plgg-sql/src/Db/model/Db.ts` - precedent for a `Box`-tagged error
  (`SqlError`) on the `Result` channel; `MigrationError` mirrors it and unifies
  with it as `E` in a `proc` chain.
- `packages/plgg-sql/src/Sql/model/Sql.ts` - precedent for a branded value
  (`SQL_BRAND`); brand `Version`/`TenantId` the same way.
- `packages/plgg/src/index.ts` - `Box`, `box`, `Option`, `some`/`none`,
  `match`, `cast`, branded-type helpers.

## Implementation Steps

1. `Version.ts` — branded string ordinal from a filename's `YYYYMMDDHHMMSS`
   prefix; `asVersion(unknown): Result<Version, MigrationError>` validating the
   14-digit shape at the boundary; ordering is a total order.
2. `Migration.ts` — `{ version: Version; name: SoftStr; up: SoftStr; down:
   Option<SoftStr>; upTransaction: Bool; downTransaction: Bool }`. `down` is
   `None` for an irreversible migration. The `*Transaction` flags carry dbmate's
   `transaction:false` directive (default `true`).
3. `MigrationDir.ts` — `ReadonlyArray<Migration>` sorted ascending by `Version`;
   the type plus a smart constructor that enforces unique, ordered versions
   (duplicate/unordered → `MigrationError`).
4. `AppliedVersion.ts` — `{ version: Version; appliedAt: SoftStr }`.
5. `SchemaMigrations.ts` — the projection of recorded history:
   `ReadonlyArray<AppliedVersion>`.
6. `Plan.ts` — `{ pending: ReadonlyArray<Migration>; applied:
   ReadonlyArray<AppliedVersion> }` for up; the single most-recent applied
   `Migration` for down (v1 = last-applied-only).
7. `Dialect.ts` — closed union `"sqlite" | "postgres" | "mysql"` plus the
   `Dialect` record `{ name; supportsTransactionalDdl: Bool }`. (The
   dialect-specific SQL builders are a separate ticket and live in the domain
   layer, not vendors.)
8. `Migrator.ts` — bound config `{ db: Db; dialect: Dialect; dir: MigrationDir }`.
9. `TenantId.ts` / `TenantDb.ts` — branded `TenantId`; `TenantDb = { id:
   TenantId; db: Db; path: SoftStr }`.
10. `MigrationError.ts` — `box("MigrationError")` with variants for parse
    failure, gap/ordering violation, irreversible-down, dialect mismatch, and
    version-shape; a `migrationError$` pattern for `match`.
11. Barrel `domain/model/index.ts`; unit tests for every caster (valid + each
    failure branch), coverage > 90%.

## Considerations

- Keep `down` strictly `Option<SoftStr>` — an empty string is not "no down".
  An irreversible migration is `None` and must fail loudly (typed) on rollback
  (`packages/plgg-db-migration/src/domain/model/Migration.ts`).
- `Dialect` holds only flags here; the per-engine SQL strings are built in the
  apply/dialect ticket so this model stays a pure vocabulary
  (`packages/plgg-db-migration/src/domain/model/Dialect.ts`).
