# U2 — Migrate `plgg-kit` from vitest to plgg-test (incl. R4 ×6 + R5 vi.mock DI)

## Overview

Migrate the `plgg-kit` package's 5 spec files from vitest to
`plgg-test`. **Ungated** package (no vitest threshold) — gets NO
`plgg-test.config.json`. Carries the two highest-effort long-tail items:
**R4** (drop 6 `test.skip` timeout args) and **R5** (the single
`vi.mock` → typed DI redesign).

**Trip Origin:** `.workaholic/trips/replace-vitest-with-plgg-test/designs/design-v2.md`
§2 (recipe), §3 **R4** + **R5**, §7 (U2).

**R4 sites (6 in this package):** `LLMs/usecase/generateObject.spec.ts`
×3 (`}, 20000)`), `LLMs/vendor/Anthropic.spec.ts`,
`LLMs/vendor/OpenAI.spec.ts`, `LLMs/vendor/Google.spec.ts` (each `}, 20000)`).
All are `test.skip` live-network integration tests. plgg-test's
`test.skip` is strictly `(name, fn) => void`, so the trailing timeout
arg is a **TS2554** that `tsc --noEmit` raises before plgg-test runs.
Drop the trailing `, 20000`. No plgg-test signature change.

**R5 site (1):** `LLMs/usecase/generateObject.spec.ts:27` —
`vi.mock("plgg", async (importOriginal) => …)` hoist-mocks the `plgg`
module's `postJson` seam for the offline `generateObject` path.
plgg-test provides no module-mock registry by design. Redesign to inject
the seam as a **typed parameter** so the offline vendor-envelope decodes
run without the network — **no `as`, no `any`, no `ts-ignore`**.
**Gate:** first confirm production `generateObject` exposes (or can
cheaply expose) an injectable seam; if it cannot, that is a small
production change to surface to the team, NOT a cast to hide behind.
