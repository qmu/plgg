This section will introduce the shared vocabulary used
across the whole plgg family:

- **Option** — absence as a value, not `null`/`undefined`.
- **Result** — errors as data, not thrown exceptions.
- **pipe / cast / proc / flow** — data-last pipeline
  composition for sync and async work.
- **match** — exhaustive folding over tagged unions.
- **Errors as data** — one error vocabulary, folded at
  the edge.
