# Demo 1 — Pane alignment

The first of plggmatic's three pillars is the
**column-oriented pane alignment system** — a screen of
column tracks stacking landmark panes.

## Run Demo

[Run demo 1](/example/demo1.html)

## What's so plggmatic

This demo builds the layout from the raw combinators —
`row`, `column`, and the landmark panes
(`navPane`/`mainPane`/`asidePane`) — composed by hand, not
through the scheduler.

A `row` holds `column` tracks; each column stacks landmark
**panes**; a track's sizing is a composed atom —
`basis("220px")` for a fixed width, `fluid` for the one
that fills the remaining space. There is no "hidden pane"
state: a pane you don't want is a pane you don't compose,
so collapsing one omits its column from the row and the
`fluid` main track reflows to fill the gap. Because the
panes are real `nav`/`main`/`aside` landmarks, the
accessibility skeleton falls out of the layout itself.

The [pane alignment](/pane-alignment) and
[multi-column renderer](/multi-column) pages explain the
system; the [workbench](/workbench) reaches the same
geometry through `schedule` + `multiColumn`. This demo
shows the primitives directly.
