# Demo 2 — Color scheme

The second of plggmatic's three pillars is the
**light/dark color scheme driven by design tokens**. This
demo proves it with the framework's own `themeToggle`
component and a grid of token swatches.

Every color in plggmatic resolves through a
`var(--pm-*)` token, and the light and dark sets of token
values are **Schemes**. The `themeToggle` flips a single
`dark` class on `<html>` through the framework's appearance
contract (`applyScheme`), and because each swatch is
painted from its token — never a baked hex — the whole
grid reschemes at once. Each swatch keeps its token name,
so nothing reads by color alone.

**[Run demo 2](/example/demo2.html)** — toggle the scheme
and watch every neutral and semantic token repaint from
the one class flip.

The [color scheme](/color-scheme) and
[design tokens](/design-tokens) pages explain the token
vocabulary; this demo makes the rescheme interactive.
