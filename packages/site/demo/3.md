# Demo 3 — Scheduler query & URL codec

The third of plggmatic's three pillars is the
**declarative scheduler**, and this demo isolates its most
quotable property: **the URL is the derived, canonical,
total codec**.

This is the smallest `schedule(declare(...))` program that
shows it — one flat, filterable collection with a `query`
box, mounted with the URL-aware `application`. You write
the collection as data; `schedule` derives the `Model`, the
`Msg` union, a pure `update`, **and** the URL codec. Then:

- typing in the filter reflects to `?q=…`;
- selecting a row reflects to `?c=…&p=…` (drilling in
  clears the filter);
- a deep link reproduces the exact slice;
- browser **back / forward** walk the selection history.

No line of hand-written parsing — the codec is derived. A
bar at the top shows the search string the model derives,
so you can watch the reflection live.

**[Run demo 3](/example/demo3.html)** — filter, select,
copy the URL to a new tab, and use back/forward.

The [workbench](/workbench) shows the full multi-level
drill-down; here one flat collection makes the codec the
whole point.
